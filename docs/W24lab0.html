Week 1: Measure a Physical Thing

We will measure a voltage which changes in a similar way as the speed of a DC motor, the temperature of an oven, and many other mechanical quantities change – as first-order linear systems.

These should be in the small parts box in your ME405 kit; if not, check the blue cabinet full of resistors. Connect them to your Nucleo with the input coming from pin C0 (marked as A5 on the circuit board) and the output going to pin B0 (marked as A3).

  2. Write a script in main.py which sets pin C0 as an output, then runs a loop in which C0 is at logic 0 for 5 seconds, then logic 1 for 5 seconds, and repeats until Ctrl-C is pressed. Save your script as square.py.

We Interrupt this Program…

For accurate timing of fast measurements, you’ll need to use timer interrupts rather than a simple loop. This is done in MicroPython by writing an interrupt callback function in Python and configuring an interrupt source to run it. Using interrupts in MicroPython is easier than doing so in assembly or C language, but there are restrictions on MicroPython interrupt code:

  • Floating point math must not be done in an interrupt callback.

  • No memory may be allocated in the callback. This includes appending or inserting items to lists or arrays. Changing values inside already allocated variables, arrays, or queues is okay.

  • Printing from within an interrupt callback is unreliable and should not be done except temporarily for debugging. Putting characters into a queue to be printed in non-interrupt code is OK.

  • Data transferred between interrupt and non-interrupt code must be protected from corruption due to the interrupt process.

MicroPython interrupt callbacks are like interrupt handlers in any other language in that they should be simple and run quickly. Interrupt callbacks can print exception messages if the following code is included near the beginning of your program:

 import micropython
 micropython.alloc_emergency_exception_buf(100)

  1. When your program takes measurements inside the interrupt callback function, how does it get those measurements to the non-interrupt part without corruption as mentioned above? One good answer is to use a queue. We have a class called cqueue.IntQueue in the ME405 version of MicroPython which does this. See the documentation of IntQueue at:

Week 2: PC Interface and Simulation

  1. Write a CPython program which runs on a laptop or desktop and provides a user interface for your step response program. The interface program should send a signal through the USB-serial port that causes the microcontroller to run a step response and print the results. The results should be read by the interface program, converted from arrays of characters to floating point numbers, put into lists, and plotted using Matplotlib. Of course, any plot must have a title and labeled axes with units, right?

    An example program has been provided which shows how to create a simple Graphical User Interface (GUI) window containing your plot and some buttons to control the test. Modify the example so that it runs a step response test and plots the result. Save a good plot for inclusion on your README page (see Deliverables below).

    Hints

    • When MicroPython uses print() it sends characters through the USB-serial port. To receive those characters on a PC, use the Serial class in the serial module. Search for pyserial on the Web for references.

    • The serial port method readline() is recommended for getting printed text from the USB-serial port. Expect to use string methods such as strip() and split() to isolate strings containing numbers and the float() function to find numbers in those strings.

    • When you disconnect Thonny from the microcontroller, MicroPython is left in “Raw REPL” mode, and your program will have a hard time talking to it. Sending a control-B character gets the microcontroller back in regular REPL mode; control-C stops any running program; control-D reboots the microcontroller and causes main.py to run.
      Some combination of these keys will probably be helpful.

    • Remember how you printed End? It will be helpful now.

    • Sometimes there are errors in transmission. Using a try–except–else structure allows us to reject bad data while keeping good data.

  2. Following the style of the example, add a second curve to your plot: A simulated VMAX (1 - e-t/RC) curve made from the values of VMAX, R, and C in your circuit. Note:

    • The ~5K resistor is not part of this equation because negligible current flows through it.

    • It’s traditional to show measured data as points and theoretical curves with lines.
